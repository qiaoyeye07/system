# 二手交易平台 五人模块分工

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 编制日期 | 2026-07-09 |
| 编制依据 | 已完成的 Service + Controller 代码结构 |
| 当前状态 | 已确认 |

## 1. 划分原则

- 每个模块有独立的 Controller、Service、Repository、实体和前端页面
- 不同模块之间代码不交叉在同一个文件中
- 共享层所有人只读，修改需通知全组

## 8. 模块依赖关系

```
模块一（认证）
  ↑ 被所有模块依赖（鉴权）

模块二（商品） ←── 模块三（订单）引用商品状态
                    模块四（聊天）引用商品 ID
                    模块五（举报）引用商品 ID

模块三（订单） ←── 模块四（评价）引用订单完成状态
                    模块五（纠纷）引用退款订单

模块四（社交） 独立，仅通过 user.id + product.id 关联

模块五（监管） 独立，引用 user.id + product.id + order.id
```
