## 1. 文档目的与范围

本文档定义二手交易平台 MVP 的全部 REST API 接口和 WebSocket 通信协议，包括路径、请求方法、请求/响应格式、鉴权规则、错误码和业务校验规则。本文档是后端 Controller 开发、前端 API 对接和接口测试的直接依据。

**技术约定：**
- 接口基础路径：`/api`
- 数据格式：`application/json`（图片上传使用 `multipart/form-data`）
- 认证方式：JWT Bearer Token（`Authorization: Bearer <token>`）
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 日期格式：`yyyy-MM-dd`

## 2. 通用规范

### 2.1 统一响应格式

所有接口返回统一的 JSON 响应结构：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `Integer` | 业务状态码，200 表示成功。 |
| `message` | `String` | 可读的提示信息。 |
| `data` | `Object/Array/null` | 响应数据。列表型返回数组，单条返回对象，无数据返回 null。 |

### 2.2 分页响应格式

支持分页的列表接口使用以下格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "content": [],
    "page": 1,
    "size": 20,
    "totalElements": 100,
    "totalPages": 5
  }
}
```

分页参数（Query String）：
- `page`：页码，从 1 开始，默认 1。
- `size`：每页数量，默认 20，最大 50。

### 2.3 错误码

| 错误码 | HTTP 状态码 | 说明 |
| --- | --- | --- |
| 200 | 200 | 操作成功。 |
| 400 | 400 | 请求参数校验失败。 |
| 401 | 401 | 未登录或 Token 过期。 |
| 403 | 403 | 权限不足。 |
| 404 | 404 | 资源不存在。 |
| 409 | 409 | 业务冲突（如用户名已存在、商品已售出）。 |
| 500 | 500 | 服务器内部错误。 |

错误响应示例：

```json
{
  "code": 400,
  "message": "用户名长度应为 3 至 50 个字符",
  "data": null
}
```

### 2.4 认证与鉴权

| 规则 | 说明 |
| --- | --- |
| Token 获取 | 登录成功返回 JWT Token，有效期 24 小时。 |
| Token 携带 | 前端存储于 localStorage，每次请求通过 `Authorization: Bearer <token>` 携带。 |
| Token 失效 | 过期或用户被禁用时，返回 401。前端拦截后清除 Token 并跳转登录页。 |
| 权限校验 | 后端 Controller 通过 `@PreAuthorize` 或 Filter 校验角色和资源归属。 |

### 2.5 通用查询参数

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `page` | int | 页码（从 1 开始），默认 1。 |
| `size` | int | 每页条数，默认 20。 |
| `sort` | String | 排序字段，格式 `field,asc\|desc`。 |
| `status` | String | 状态筛选。 |

## 3. 认证接口（/api/auth）

### 3.1 用户注册

```
POST /api/auth/register
```

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "12345678",
  "confirmPassword": "12345678"
}
```

| 字段 | 类型 | 必填 | 约束 |
| --- | --- | --- | --- |
| `username` | String | 是 | 去首尾空白后 3-50 字符；允许中文、英文、数字、下划线、连字符。 |
| `password` | String | 是 | 8-128 字符。 |
| `confirmPassword` | String | 是 | 必须与 password 完全一致。 |

**成功响应（200）：**

```json
{
  "code": 200,
  "message": "注册成功，请登录",
  "data": null
}
```

**业务校验：**
- 用户名已存在 → 409："用户名已被使用"
- 密码与确认密码不一致 → 400："两次输入的密码不一致"
- 字段为空或长度/格式不合法 → 400：对应字段提示

---

### 3.2 用户登录

```
POST /api/auth/login
```

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "12345678"
}
```

**成功响应（200）：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "user": {
      "id": 1,
      "username": "zhangsan",
      "role": "USER"
    }
  }
}
```

**失败响应：**
- 用户名不存在、密码错误或账号被禁用 → 401："用户名或密码错误，或账号不可用"

---

### 3.3 退出登录

```
POST /api/auth/logout
Authorization: Bearer <token>
```

**成功响应（200）：**

```json
{
  "code": 200,
  "message": "已退出登录",
  "data": null
}
```

> MVP 中退出为客户端行为（清除 Token），服务端可选择性维护 Token 黑名单。

### 6.4 发布商品

```
POST /api/products
Authorization: Bearer <token>
Content-Type: multipart/form-data
```

权限：已登录且启用的普通用户。

**表单字段：**

| 字段 | 类型 | 必填 | 约束 |
| --- | --- | --- | --- |
| `title` | String | 是 | 1-100 字符。 |
| `price` | BigDecimal | 是 | 正数，最多两位小数，最大 999999.99。 |
| `description` | String | 是 | 1-4000 字符。 |
| `categoryId` | Long | 是 | 必须为有效分类。 |
| `condition` | String | 是 | 枚举：NEW / LIKE_NEW / USED。 |
| `tradeType` | String | 是 | 枚举：PICKUP / EXPRESS / BOTH。 |
| `tradeMode` | String | 是 | 枚举：SELL / SWAP / BOTH。 |
| `location` | String | 是 | 1-50 字符。 |
| `tags` | String | 否 | 逗号分隔，最多 5 个，每个 ≤20 字符。 |
| `images` | File[] | 否 | 最多 3 张，单张 ≤5MB，支持 jpg/png/gif。 |

**成功响应（200）：**

```json
{
  "code": 200,
  "message": "商品发布成功，当前状态为在售",
  "data": { "id": 10 }
}
```

**校验失败：**
- 分类不存在或已停用 → 400
- 图片超量或超大 → 400："商品图片最多 3 张，每张不超过 5MB"

---

### 6.5 下架商品
