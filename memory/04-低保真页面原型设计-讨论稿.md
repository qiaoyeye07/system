## 1. 设计目的

本文档用于在进入代码开发前，以低保真线框方式帮助确认页面结构、信息优先级、操作入口和关键状态表现。本文档不定义最终视觉风格、颜色、字体、图标、响应式细节或前端实现方式。

低保真设计遵循以下边界：

- 不扩大 MVP 功能范围。
- 页面只呈现当前角色可理解或可执行的主要操作。
- 后端权限校验仍是最终安全边界。
- 采用线性流程型页面结构：多页面、单页面单主任务、顶部页头与角色导航、列表进入详情、详情集中展示状态与可执行操作。
- 若本文档与已确认的 PRD、SRS、页面与交互说明发生冲突，以已确认正式基线文档为准。

## 2. 全局页面框架

登录后的用户端和管理端共用基础框架，但导航项按角色变化。

```text
┌──────────────────────────────────────────────────────────────┐
│ 二手交易平台                              用户名 ｜ 退出      │
├──────────────────────────────────────────────────────────────┤
│ 导航：当前角色可访问模块                                     │
├──────────────────────────────────────────────────────────────┤
│ 页面标题                                                     │
│ 操作反馈区：成功 / 错误 / 提示                                │
│                                                              │
│ 当前页面主体内容                                             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.1 普通用户导航

```text
首页 | 发布商品 | 我的商品 | 我的订单 | 聊天 | 个人主页 | 退出
```

### 2.2 管理员导航

```text
首页 | 发布商品 | 聊天 | 用户管理 | 分类管理 | 纠纷处理 | 退出
```

## 6. 关键流程低保真流转

### 6.1 商品发布到购买

```text
登录页 → 注册页 → 登录页 → 首页
                              ↓
                        发布商品页 → 商品详情页（在售）
                              ↑
                         我的商品页
                              
首页 / 分类页 / 搜索 → 商品详情页 → 聊天页（联系卖家）
                            ↓
                      订单详情页（待付款）
```

### 6.2 担保交易流程

```text
订单详情（待付款）
  → 模拟付款 → 订单详情（已付款）
  → 卖家发货 → 订单详情（已发货）
  → 买家收货 → 订单详情（已收货）
  → 自动完成 → 订单详情（已完成）
  → 评价对方
```

### 6.3 退款纠纷流程

```text
订单详情（已收货）
  → 买家申请退款 → 卖家收到退款申请
  → 卖家同意 → 订单取消，商品恢复在售
  → 卖家拒绝 → 买家申请管理员介入
  → 管理员查看纠纷详情 → 裁定（同意退款 / 维持原状）
```

### 6.4 管理员初始化

```text
首次启动 → /api/init 创建管理员
  → 管理员登录
  → 分类管理页：添加商品分类
  → 平台就绪，等待用户注册
```

### 6.5 以物易物流程

```text
商品详情页（可交换商品）
  → 发起交换页：选择自己的在售商品 → 提交
  → 交换订单详情页（待确认）
  → 卖家查看：同意交换 / 拒绝交换
  → 同意后：交换订单详情页（已确认）
  → 双方各自发货 → 双方已发货
  → 双方各自收货 → 已完成
  → 双方互评
```
